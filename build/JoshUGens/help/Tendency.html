Tendency creates dynamic random number distributions


Inherits from: Object


Tendency stores data for creating random values where the parameters for the random number generation occur over time. 


Some Important Issues Regarding Tendency 


Though the parameters 'high' and 'low' (see *new method below) are gnerally used to describe bounds over time, in certain distributions, these actual parameters may control other aspects of a random distribution. See the descriptions of the \cauchy, \gauss and \poisson distributions for more details.


Creation / Class Methods


*new (high, low, parA, parB)

Creates a new Tendency object. Any of the values below may be a SimpleNumber, Env or Function. If a Function is used, the current time will be passed into the Function.

high - For the most common usage, describes the upper bounds of the distribution. Default value is 1.0. 

low - For the most common usage, describes the upper bounds of the distribution.. Default value is 0.0.

parA - For some distributions, the control of a specific parameter. Default value is 0.1

parB -  For some distributions, the control of a specific parameter. Default value is 0.1.

// inline example

g = Tendency.new(1.0, 0.0);

Using Tendency


The basic usage of Tendency will be through the 'at' method. A random value wll be calculated according to the  state of 'high' and 'low' at 'time' for a specific 'dist'.

at (time, dist)

Short prose description of method.

time - The time to calculate a random number for. Default value is 0.0.

dist - The distribution to use. Default value is \uniform. The kind of distribution may change the meaning of some of the initial args (e.g., 'high' may be treated as a 'mean').

(

// basic Example.

g = Tendency.new(

Env([1.0, 0.5, 1.0], [0.5, 0.5]), 

Env([0.9, 0.0, 0.9], [0.8, 0.2], [-10, 5]));

n = 500; // get 500 samples

o = n.collect({arg i; i = i / n; g.at(i)}); 

o.plot(discrete: true);

)


// using different distributions

// mean (or band pass) distribution

o = n.collect({arg i; i = i / n; g.at(i, \meanRand)});

o.plot(discrete: true);


// low pass distribution

o = n.collect({arg i; i = i / n; g.at(i, \lpRand)});

o.plot(discrete: true);


// hi pass distribution

o = n.collect({arg i; i = i / n; g.at(i, \hpRand)});

o.plot(discrete: true);


// exponential distribution

o = n.collect({arg i; i = i / n; g.at(i, \expRand)});

o.plot(discrete: true);


/* The following distributions are special cases! high and low do not act the same as in previous cases */


// cauchy - with this distribution, 'high' is the mean of the distribution

// and 'low' is the spread


g = Tendency.new(

Env([1.0, 5.0], [1]), // move the mean

0.1);

o = n.collect({arg i; i = i / n; g.at(i, \cauchy)});

o.plot(discrete: true);


// gaussian - 'high' is the mean, 'low' is the deviation

g = Tendency.new(

Env([1.0, 5.0], [1]), // move the mean

0.1);

o = n.collect({arg i; i = i / n; g.at(i, \gauss)});

o.plot(discrete: true);


// poisson - 'high' is the mean

g = Tendency.new(

Env([1.0, 5.0], [1])); // move the mean

o = n.collect({arg i; i = i / n; g.at(i, \poisson)});

o.plot(discrete: true);

/* \beta uses the parA and parB arguments to describe where the likelihood that a random value will occur near high or low. For example, if parA < 1, values are more likely to 

occur close to high. if parA is 1, the distribution is uniform. > 1, and you get a bounded gaussian like distribution */

g = Tendency.new(1.0, 0.0, Env([0.01, 2.0], [1], \sin), Env([0.01, 2.0], [1], \sin));

o = n.collect({arg i; i = i / n; g.at(i, \betaRand)});

o.plot(discrete: true);



asStream (dist)

SynthDef(\tendency, {arg freq;

Out.ar(0, SinOsc.ar(freq, 0, XLine.kr(0.1, 0.0001, 1, doneAction: 2)));

}).send(s);

(

Routine.run({

g = Tendency.new(1760.0, 440.0, Env([0.01, 2.0], [10], \sin), Env([0.01, 2.0], [10], \sin)).asStream(\betaRand);

t = Main.elapsedTime;

while({

Synth(\tendency, [\freq, g.next]);

0.02.wait;

(Main.elapsedTime - t) < 10;

})

})
